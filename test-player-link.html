LibreTV 播放器测试链接

问题诊断

当前播放器显示"正在加载视频..."的原因是缺少必要的URL参数。播放器需要以下参数：

  • url - 视频文件的直接链接
  • title - 视频标题
  • source - 视频源标识

当前问题

直接访问 player.html 页面而不传递任何参数会导致播放器无法加载视频，因为没有视频URL可供播放。

测试链接

测试1: 标准MP4视频播放
URL: player.html?url=https://commondatastorage.googleapis.com/gtv-videos-bucket/sample/BigBuckBunny.mp4&title=大雄兔测试视频&source=test
测试2: HLS流媒体播放
URL: player.html?url=https://demo.unified-streaming.com/k8s/features/stable/video/tears-of-steel/tears-of-steel.ism/.m3u8&title=HLS测试视频&source=test
测试3: 完整参数测试
URL: player.html?url=https://sample-videos.com/zip/10/mp4/SampleVideo_1280x720_1mb.mp4&title=示例视频&source=test&index=0&position=0

解决方案

要正常使用播放器，需要：

  1. 确保通过正确的URL参数访问播放器页面
  2. 提供有效的视频URL
  3. 确保视频URL可以正常访问（没有CORS限制）
  4. 检查网络连接是否正常

调试步骤

  1. 点击上面的测试链接验证播放器功能
  2. 如果测试视频可以播放，说明播放器功能正常
  3. 如果测试视频无法播放，检查浏览器控制台错误信息
  4. 确认实际使用时传递了正确的URL参数
